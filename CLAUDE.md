# CLAUDE.md - Survey Reports App

## Project Overview

A Next.js web application for generating AI-powered analysis reports from Excel/CSV survey data. Built for Spanish-speaking users with client branding, automated report generation using Claude AI, and multiple export formats (PDF, Excel, HTML).

## Tech Stack

- **Framework:** Next.js 16 (App Router) with React 19, TypeScript (strict)
- **Styling:** Tailwind CSS 4
- **Auth & DB:** Supabase (email OTP magic links, PostgreSQL, storage for logos)
- **AI:** Anthropic Claude API (`@anthropic-ai/sdk`) - Claude Sonnet 4
- **File Parsing:** `xlsx` (Excel) + `papaparse` (CSV)
- **Charts:** Chart.js + react-chartjs-2
- **Exports:** XLSX for Excel export, HTML with `@page` CSS for PDF (via browser print)

## Commands

```bash
npm run dev       # Start development server
npm run build     # Production build
npm run start     # Start production server
npm run lint      # Run ESLint
```

No test framework is configured.

## Database Schema (Supabase/PostgreSQL)

- **emitter_settings** - Company branding (name, logo, phones, emails, web, linkedin, addresses)
- **clients** - Customer companies (name, logo_url, notes)
- **reports** - Generated reports (title, period, report_type, ai_analysis JSON, report_html, report_data JSON)
- **access_logs** - Audit trail (user_email, action, path, user_agent)
- **storage bucket: logos** - Logo file uploads

RLS is enabled; all authenticated users share full read/write access.

## Key Architecture

### Authentication Flow
Email OTP via Supabase -> magic link -> `/auth/callback` exchanges code for session -> middleware protects `(app)/*` routes.

### Report Generation Pipeline
1. User uploads Excel/CSV -> `parser.ts` extracts data
2. `stats.ts` builds column statistics (type, top values, unique counts)
3. Stats sent to `/api/analyze` -> Claude returns `AIAnalysis` JSON
4. User selects questions, chart types, and report format
5. `processor.ts` calculates frequencies, percentages, funnel stages
6. Report HTML generated by `charts-html.ts`, `table-html.ts`, or `flowchart-html.ts`
7. Logos embedded as base64 data URIs
8. Report saved to Supabase; exportable as PDF/Excel/HTML

### Report Types
- **Charts** - One chart per question (pie, doughnut, bar, horizontalBar)
- **Table** - Summary boxes + hierarchical data table with funnel
- **Flowchart** - Node tree with connectors showing data flow

### Claude API Key
Stored in `localStorage` only (never on server). Passed from client to `/api/analyze` per request.

### API Cost Tracking
Token usage calculated client-side: input at $3/1M tokens, output at $15/1M tokens. Displayed in the report wizard.

## Conventions

### Language
- **UI text is in Spanish** (labels, messages, alerts, placeholders)
- **Code is in English** (variables, functions, comments)

### Naming
- `camelCase` for variables and functions
- `PascalCase` for components and types
- `UPPER_CASE` for constants (e.g., `DEFAULT_STYLE`)
- Column references use Excel-style letters (A, B, C... AA, AB...)

### Styling
- Tailwind CSS utility classes for components
- Inline CSS for report HTML generation (must be self-contained for PDF)
- Primary color palette: greens (`#53860F`, `#2D4A06`, `#7AB317`)
- Responsive breakpoints: `sm`, `lg`

### Patterns
- `"use client"` directive on interactive components
- `useState` / `useCallback` / `useEffect` for state (no external state library)
- Server-side Supabase client via `createClient()` from `lib/supabase/server.ts`
- Browser Supabase client via `createClient()` from `lib/supabase/client.ts`
- All DB functions are async, use try/catch, return data or null on error
- Access logging with 30-minute dedup cooldown for page views
- Path alias: `@/*` maps to project root

### Error Handling
- Try/catch in all async functions
- User-facing errors shown via `alert()` or inline messages (in Spanish)
- API routes return structured JSON errors with appropriate HTTP status codes

### Report HTML Generation
- All report HTML must be fully self-contained (inline styles, base64 images)
- Uses `@page` CSS rules for print/PDF layout
- Supports A4 landscape/portrait orientation
- Chart images rendered server-side via Chart.js canvas -> base64 PNG

## Environment Variables

Required in `.env.local`:
- `NEXT_PUBLIC_SUPABASE_URL` - Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Supabase anonymous key

The Claude API key is provided by the user at runtime via the Settings page and stored in browser localStorage.
